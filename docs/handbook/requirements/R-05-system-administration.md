# 需求说明：系统管理

## 1. 目标

维护组织架构、账号、角色、API 权限定义、**角色-权限授权**、动态菜单、数据字典、登录/操作日志、封禁 IP、K8s 三元策略下发界面。

## 2. 功能列表

| 模块 | 子功能 |
|------|--------|
| 组织架构 | 部门树、用户归属 |
| 账号管理 | 用户 CRUD、角色分配、导入导出 |
| 角色 / API / 授权 | 权限元数据 + Casbin 策略行 |
| 菜单管理 | 树形菜单、component 与前端懒加载约定 |
| 数据字典 | 多类型键值，告警/邮件等可字典优先 |
| 注册审核 | 待审核列表、通过/拒绝 |
| 日志与审计 | 登录日志、操作历史、导出 |
| 封禁 IP | 列表与解封 |
| K8s 三元策略 | 按角色下发集群范围读写策略 |

## 3. 注意事项

- `super-admin` 在中间件层直接放行，但仍应通过种子保证权限表完整。
- 菜单 `component` 字段对应 `web/src/pages/**/*-page.tsx` 的 loader 名。
- 授权变更后需 `SyncUserRoles`（服务内已调用）使 Casbin 生效。
